Aprendizado

Ao iniciar o curso de Desenvolvimento Web na trybe notamos que o rítimo é bastante acelerado e por isso devemos dedicar uma quantidade de tempo maior ainda. Na primeira semana já iniciamos com a familiarização do ambiente de Desenvolvimento que é exigido: instalação do S.O Ubuntu, configuração do terminal, criação do repositório no GitHub e introdução aos comandos do Git. Já na segunda semana iniciamos com o tão esperado HTML e seu parceiro CSS. Descobrimos como integrar ambos em um só projeto para que funcionem juntos. No mais, tivemos uma introdução no ambiente de desenvolvimento e suas ferramentas que nos acompanharão em nossas carreiras durante toda a vida. A cada momento nos é apresentado novos conhecimentos que nos levam a novos desafios, e em busca da resolução de novos problemas nos construímos como desenvolvedores. Tão logo se encerra o dia-a-dia na Trybe, bem como seus desafios diários, buscamos novas formas de implementar os códigos e revisar os que já foram implementados. Esse busca por uma maneira ou modo de fazer melhor, de fazer diferente, nos leva ao desenvolvimento das hards skills e o aprimoramento como Web Desenvolvedor. E ao aprimorar-se fazemos contato direto com o código limpo, tangível, editável e mais que nunca enxuto. Isso é o que se é esperado por um navegador ao interpretar os códigos html e css: código enxuto, limpo e organizado. Ao acessar uma página web através de um navegador, ele é capaz de interpretar o código HTML e renderizá-lo de forma compreensível para o usuário final, exibindo textos, botões etc. com as configurações definidas por meio das diversas tags que essa linguagem dispõe. Atualmente a HTML encontra-se na versão 5 e é padronizada pelo W3C (World Wide Web Consortium), uma organização internacional responsável por estabelecer padrões para a internet, como a linguagem XML, CSS e o SOAP.
